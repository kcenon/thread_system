Detailed Description

Manages a group of thread_worker instances and a shared job_queue for concurrent job processing.

The thread_pool class provides an interface to:

  • Maintain a shared job_queue
  • Maintain multiple thread_worker objects
  • Enqueue jobs (job) into the shared queue
  • Start/stop all worker threads as a group

This class inherits from std::enable_shared_from_this, allowing you to safely retrieve a std::shared_ptr<thread_pool> from member functions via get_ptr().

Typical Usage

  1. Create a thread_pool (usually with std::make_shared).
  2. Optionally create and enqueue additional thread_worker objects (if you want specialized behaviors).
  3. Call start() to launch all workers, enabling them to process jobs in the job_queue.
  4. Enqueue job objects into the shared queue as needed.
  5. Eventually call stop() to allow all threads to finish current tasks (or stop immediately).
