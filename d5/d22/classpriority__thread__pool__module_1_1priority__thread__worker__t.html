Detailed Description

template<typename priority_type = job_priorities>
class priority_thread_pool_module::priority_thread_worker_t< priority_type >

A template-based worker thread class that processes jobs from a priority job queue.

This class extends the functionality of thread_base by continually retrieving and executing jobs from a priority_job_queue_t. Each worker can be configured to handle specific priority levels, allowing for flexible job distribution among multiple workers.

Template Parameters
priority_typeA type that represents the priority levels. Often an enumeration (e.g., job_priorities). By default, it is set to job_priorities.

Usage Example

auto worker = std::make_shared<priority_thread_worker_t<job_priorities>>(
std::vector<job_priorities>{job_priorities::HIGH, job_priorities::MEDIUM},
true
);
auto queue = std::make_shared<priority_job_queue_t<job_priorities>>();
worker->set_job_queue(queue);
// Start the worker thread
worker->start();
// Enqueue some jobs
queue->push_job(
// job lambda or functor
, job_priorities::HIGH
);
// Stop and join the worker once done
*worker->stop();
*worker->join();

Thread Safety

The worker is designed to run its own thread that continuously checks the queue for new jobs. Any shared data structures should be either thread-safe or externally synchronized.

Executes pending work by processing one job at a time.

This method pops a job from the queue (if available) and executes it. The method is repeatedly called in the worker thread’s loop until the worker is stopped or no more jobs are available.

Returns
An optional string that may hold a status or error description:
  • If std::nullopt is returned, it implies the job was processed without issues or there was no job to process.
  • If a non-empty string is returned, it typically contains debug or error information that can be logged or reported.
Note
Derived classes can override this method if custom processing logic is needed for job handling or error management.

Assigns a priority job queue to this worker.

Once a queue is set, the worker will continuously retrieve jobs that match the configured priority levels. If a different queue is assigned later, the worker will switch to that queue for subsequent job retrieval.

Parameters
job_queueA shared pointer to a priority_job_queue_t instance from which this worker will fetch jobs.
Note
It is the responsibility of the caller to ensure the job_queue remains valid for the lifespan of this worker.
