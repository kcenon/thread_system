Detailed Description

A specialized job class that encapsulates a user-defined callback.

The callback_job class provides an interface for executing a user-supplied function within a job queue. The callback function returns an std::optional<std::string>, where a value (string) typically indicates an error message and std::nullopt signifies a successful operation.

Usage Example:

auto job = std::make_shared<callback_job>(
[]() -> std::optional<std::string> {
// Perform some work here...
bool success = do_some_work();
if (!success) {
return std::string{"Work failed due to ..."};
}
return std::nullopt; // No error => success
},
"example_callback_job"
);
// Submit job to a queue or execute it directly...

Returns
std::optional<std::string>
  • If std::nullopt, the job is considered successful.
  • If a std::string is returned, it typically represents an error message or reason for failure.

This method is called internally by the job queue or any mechanism that processes job instances. In user code, you generally won't call do_work directly unless you are bypassing a job queue and want to execute the job on demand.
